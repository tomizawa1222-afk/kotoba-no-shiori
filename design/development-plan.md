# 開発計画書

## 1. 開発ロードマップ

### Phase 1: MVP (Minimum Viable Product)
*   **目標**: 基本的なタスク管理機能を実装し、チーム内でドッグフーディングを開始する。
*   **期間**: 2-3週間
*   **スコープ**:
    *   ユーザー認証・チーム管理
    *   タスクのCRUD・カンバンボード表示
    *   Supabase Realtime同期

### Phase 2: 差別化機能の実装
*   **目標**: 「Owaraseru」独自の価値である集中機能を実装し、正式リリースを目指す。
*   **期間**: 2週間
*   **スコープ**:
    *   集中モード（タイマー、フィルタリング）
    *   プッシュ通知（期限切れリマインダー）
    *   UI/UXのブラッシュアップ（アニメーションなど）

### Phase 3: 運用・改善
*   **目標**: ユーザーフィードバックに基づく改善。
*   **スコープ**:
    *   バグ修正
    *   パフォーマンスチューニング
    *   （要望があれば）機能追加検討

## 3. リスク管理と対策

| リスク項目 | 優先度 | 対策 |
| :--- | :--- | :--- |
| **Supabase Realtime接続制限** | 高 | 接続切断時の自動再接続ロジックを実装し、UXへの影響を最小化する。将来的なProプラン移行を想定しておく。 |
| **プッシュ通知の許可拒否** | 中 | 初回アクセス時にいきなり許可を求めず、ユーザーがメリットを理解したタイミング（設定画面や初回の期限設定時）で許可を促すフローにする。 |
| **シンプルさの欠如** | 中 | 機能追加要望が出ても、「やらないことリスト」に基づき慎重に判断する。デザインレビューで「要素が多すぎないか」を常にチェックする。 |
