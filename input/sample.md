# タスク管理アプリ「Owaraseru」 ビジネス要件定義書 (Ver. 0.1)

## 1. プロジェクトの背景と目的

### 1.1. 背景
世の中には多機能なタスク管理ツールが溢れているが、その多くが「ツールを管理するためのタスク」を新たに生み出している。高機能すぎて使いこなせない、通知が多すぎて集中できない、設定が複雑で導入が進まない、といった問題が散見される。

### 1.2. 目的
本プロジェクトの目的は、「タスクを終わらせること」だけに焦点を当てた、**究極にシンプルなタスク管理アプリ**を提供することである。
ユーザー（主に我々開発チーム）を「終わらないタスク」と「無限の通知地獄」から解放し、本質的な業務への集中と、結果としての「定時退社」の実現を支援する。

## 2. ビジネスゴール

* **ゴール1 (可視化):** プロジェクトマネージャーが「進捗どう？」とメンバーに尋ねる回数を激減させる。
* **ゴール2 (明確化):** 「これ誰のタスク？」問題、「いつまでだっけ？」問題を撲滅する。
* **ゴール3 (効率化):** アプリ操作にかかる時間を最小限にし、実際の作業時間を最大化する。

## 3. ターゲットユーザー

| ペルソナ | 特徴 | 求めること (要求) |
| :--- | :--- | :--- |
| **疲弊したPM** | 複数案件を掛け持ち。スプレッドシートとチャットを往復。 | 「誰が、何を、いつまでに」が一目でわかるダッシュボード。 |
| **集中したい作業者** | 割り込みタスクが多い。ツール通知に集中を削がれる。 | 自分のタスクだけを静かに確認したい。「今は話しかけるな」機能。 |
| **状況を知りたい上司** | 現場の詳細は不要。 | 「ヤバい（遅延してる）」プロジェクトだけ赤字で教えてくれれば良い。 |

## 4. 主要なビジネス要件（スコープ）

本アプリは、以下の機能要件を**必須（Must）**とし、それ以外の機能は**原則として実装しない（Won't）**。

### 4.1. 必須要件 (Must Have)

* **要件1: 究極のタスク登録**
    * **要求:** タスク登録項目は「タスク名」「担当者」「期限」の3つのみを必須とする。（備考欄は任意）
* **要件2: シンプルなカンバンボード**
    * **要求:** ステータスは「未着手」「作業中」「完了」の3つに固定する。（「レビュー中」などは不要）
    * 
* **要件3: 控えめなリマインダー**
    * **要求:** 期限日の朝に一度だけ、担当者に通知する。PMが「進捗どう？」と聞く代わり。
* **要件4: 「集中モード」**
    * **要求:** オンにすると、一定時間（例: 25分）全ての通知を停止し、自分のタスク一覧だけに集中できる。

### 4.2. やらないこと (Won't Have)

* 複雑なガントチャート
* タスクの親子関係（依存関係）
* 工数見積もり、実績管理
* チャット機能（既存のツールを使えば良い）

## 5. 成功の指標 (KPI)

* **指標1:** ユーザーあたりの「完了」タスク数 / 日
* **指標2:** 期限切れタスクの発生率の低下
* **指標3:** （夢）導入チームの平均残業時間
